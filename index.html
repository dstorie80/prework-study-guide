HTML

  • HEAD
    • Head - Contains the metadata to render the page
    • Title - Defines the title of the webpage (shown in the browser)
    • Style - Defines the CSS styles associated w/ webiste
    • Base - Defines the base URL for a webpage
    • Links - Connects an external resource to the HTML doc
    • Meta - Defines metadata sucha as a character set description, keywords, author and/or viewpoint
    • Script- Embeds client side scripts in the HTML
  • BODY
    • h1 - Headers
      • h1
      • h2
      • h3
    • p - Paragraphs
    • ul - Unordered lists
    • li - List items
    • img - Information about images
      • src - Where the image comes from
      • alt - Gives alternative description of the image
      • width - Image width
      • height - Image height
    • br - Creates a line break between elements
    • a - Creates an anchor (links) to the same webpage or other websites
    • href - points the url for the link
      • (ex: < a href: (img url) < /a>)

    GIT

  • GITHub
    • Repo - Short for repository
    • Issues - User story and descirptions used to describe what needs to be addressed for webpage
    • Code - Allows user to download repo files using git bash
  • GITBash commands
    • ls - Lists files and folders in current directory
    • cd - Changes the directory
    • cd .. - Moves the directory back one level
    • pwd - Prints the current directory
    • mkdir - Creates a new folder in the current directory
    • touch - Creates a new file in the current direcory
      • touch sample_code.txt (creates a new notepad doc named sample_code)
    • start (file name) - Opens the file in the current directory
      • start sample_code.txt (opens the notepad file named sample_code as long as it exists in the current directory)
  • GitHub Commands
    • git clone (url of repo) - Clones the repo to the local machine
    • git remote -v - Shows the fetch and push urls for the repo
    • code . - Opens VSCode for the cloned repo
    • git status - Sees the current branch of the repo
    • git checkout - b (branch name) - Creates a new branch of the repo
    • git add -A - Records changes to files in the current repo
    • git commit -m - Commits changes to the repo branch and adds a message describing the changes
    • git pull origin main - Checks status of change/commits
    • git push origin (name of branch) - Pushes changes to branch
    • git checkout main - Returns to the main branch of the repo
